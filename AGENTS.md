# AGENTS

## 项目目标

- 这是一个本地优先的长篇小说创作 MCP Server。
- 第一版只负责存储、检索、连续性检查和写作上下文构建。
- 第一版不接 OpenAI API、Claude API 或 Web 前端。

## 代码风格

- 使用 TypeScript 严格模式。
- 默认保持 ASCII；只有已有中文内容或业务文本必须中文时再写中文。
- 业务逻辑优先放在 `src/services`，MCP 暴露层保持薄封装。
- 所有新工具都应返回稳定 JSON 结构，不要直接抛裸错误给客户端。

## 数据库兼容性

- 不允许破坏既有 SQLite schema 兼容性。
- 任何表结构变更都必须更新 `src/db/migrations.ts`。
- 新增检索或索引逻辑时，优先考虑旧数据兼容和迁移成本。

## 工具与输入校验

- 所有 MCP tools 必须使用 Zod 做参数校验。
- 新增 tool 时，优先在 service 层实现业务，再在 `src/tools` 暴露。
- 新增 resource 或 prompt 时，确保内容可以直接被客户端消费，不依赖隐式上下文。

## 文档要求

- 新增核心功能后，必须同步更新 `README.md`。
- 如果变更了命令、环境变量、资源 URI、prompt 名称或 tool 名称，必须同步更新 README。

## 依赖约束

- 不要硬编码 API Key、Token 或私密路径。
- 不要引入没有明显收益的大型依赖。
- 能用 Node 标准库、SQLite 和现有工具解决的问题，不要额外加框架。

## 测试要求

- 核心工具路径必须有 Vitest 覆盖。
- 至少保证项目、人物、世界观、章节、伏笔、上下文构建和连续性检查可测。
- 修改连续性规则、章节保存或检索逻辑时，优先补回归测试。
